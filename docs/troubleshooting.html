Troubleshooting

Verify first. Reconcile second. Repair from there.

Use the built-in verification and reconciliation commands first whenever the panel, nginx config, runtimes, or managed host resources drift away from expected state.

Core Recovery Commands 
sudo /home/deploycp/core/bin/deploycp verify-host
sudo /home/deploycp/core/bin/deploycp reconcile-managed
Step 1

Run host verification.

sudo /home/deploycp/core/bin/deploycp verify-host

The command validates root access, required binaries, service paths, configured directories, firewall backend support, and the configuration needed for local managed DB operations.

Step 2

Re-apply managed state.

sudo /home/deploycp/core/bin/deploycp reconcile-managed

This re-renders and re-applies generated config, service units, cron entries, firewall rules, Redis instances, FTP configuration, and Varnish includes from the saved metadata.

Core Checks

Check the panel service and web stack.

DeployCP service

sudo systemctl status deploycp --no-pager

Panel journal

sudo journalctl -u deploycp -n 200 --no-pager

Nginx validation

sudo nginx -t

Bootstrap host

sudo /home/deploycp/core/bin/deploycp bootstrap-host
Common Cases

What to check when a live action fails.

Managed DB creation fails

Check the admin credentials in /home/deploycp/core/.env for MariaDB and PostgreSQL before retrying the action.

Nginx updates do not go live

Run sudo nginx -t. DeployCP aborts reload when config validation fails.

Runtime installation fails

Fix the host package or runtime availability issue first, then rerun reconciliation so generated platform state stays aligned.

Varnish looks configured but traffic is unchanged

Confirm that live traffic really passes through Varnish before nginx. Valid generated VCL alone does not mean active caching.

Recommended Order

Use this recovery sequence on production hosts.

01

Inspect service state

Start with systemctl status deploycp and the journal.

02

Run verification

Use deploycp verify-host to surface broken prerequisites and env issues.

03

Fix missing prerequisites

Correct binary paths, admin passwords, service availability, or env values on the host.

04

Reconcile managed state

Run deploycp reconcile-managed and validate nginx plus the affected app or platform service unit.
