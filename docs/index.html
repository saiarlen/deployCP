DeployCP Linux Docs

Production hosting docs with a real one-command install.

DeployCP is built to install fast, wire real Linux resources, and keep platform operations behind the existing panel flow. These docs focus on what operators need first: install, verify, run, recover.

Start Here

The shortest path from fresh server to live panel.

01

Run the bootstrap installer

Use the remote installer on a clean Linux server. It resolves the latest release, pulls the correct artifact, and runs the packaged installer automatically.

curl -fsSL https://raw.githubusercontent.com/saiarlen/deployCP/main/scripts/linux/install-remote.sh | sudo bash
02

Verify the host after bootstrap

Check the systemd unit and run the built-in verification command before creating live platforms.

sudo systemctl status deploycp --no-pager
sudo /home/deploycp/core/bin/deploycp verify-host
03

Set production values

Update /home/deploycp/core/.env with the real base URL, bootstrap admin credentials, and any local DB admin passwords you want DeployCP to use for managed provisioning.

Managed Scope

What DeployCP actually changes on the host.

Platform resources

Creates platform folders, site users, runtime env files, scoped logs, cron wrappers, and app process units under the managed directory tree.

Web stack

Writes nginx config, validates before reload, issues or maps certificates, and keeps platform logs tied to their actual host files.

Data and edge services

Handles managed DB users and databases, FTP users, Redis instances, Varnish includes, and firewall rules from the panel flows.

Lifecycle

Install, update, reconcile, verify, and uninstall are all packaged with the release so the operator path stays consistent.
