Managed Operations

What the panel writes, starts, reloads, and cleans up on Linux.

DeployCP keeps the existing panel flow, but the work behind it is real host behavior: scoped users, directories, runtime selection, config generation, service units, firewall rules, and lifecycle cleanup.

Platform Scope

Per-platform resources are host-backed.

Site users

Platform site users are real Linux users with scoped home and folder access. SSH and runtime defaults are applied through the platform environment.

Platform folders

Each platform gets a real directory tree for web files, logs, temp state, runtime metadata, and generated process files.

Cleanup

Platform deletion removes scoped Linux resources in cleanup order rather than only deleting SQLite rows.

Runtime Strategy

Runtime version selection is per platform, not global guesswork.

01

Catalog changes install or remove runtimes

Node, PHP, Python, and Go version actions call the runtime installer and keep side-by-side versions when supported.

02

Platform defaults are written into runtime env files

Each platform gets .deploycp/runtime.env, which is then sourced by SSH shells, cron wrappers, and generated service units.

03

In-use versions are protected

DeployCP blocks runtime removal when platforms still depend on that version.

Web And Edge

Nginx, SSL, firewall, and Varnish are all wired to the host.

Nginx

Writes real config, validates it before reload, supports redirects, ACME challenge paths, IP restrictions, auth, and managed platform log files.

SSL

Uses Let's Encrypt when Certbot is available and supports imported or self-signed certificates stored in managed SSL storage.

Firewall

Applies real host rules from the settings page using ufw, firewall-cmd, or iptables.

Varnish

Generates per-site VCL fragments, validates them with varnishd -C, and reloads the configured Varnish service.

Services And Data

Panel actions map to service-backed resources.

AreaHost behavior
system servicesDetects installed state, installs packages when needed for supported services, and runs start/stop/restart/status via systemd.
MariaDB / PostgreSQLCreates and drops managed local databases and users when admin credentials are configured.
FTPCreates real Linux FTP users with a no-login shell and updates managed ProFTPD config snippets.
RedisCreates dedicated Redis instances as systemd units and keeps DB numbering and ports unique where applicable.
cronWrites wrapper scripts plus /etc/cron.d entries and removes them during cleanup.
Logs And Delete Flow

Logs stay mapped to real files, and delete cleans real resources.

Logs: Access, error, and app logs are read from actual platform log paths, not fake sample output.
Delete: Platform deletion removes scoped users, folders, nginx config, SSL assets, cron entries, FTP users, managed DB resources, Redis units, and generated runtime state.
