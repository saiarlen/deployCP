Install Guide

Install or update DeployCP from one remote command.

The public path is the bootstrap installer. It picks the correct Linux release bundle, stages the binary, runs the packaged install or update script, and leaves the service ready for verification.

Latest Release 
curl -fsSL https://raw.githubusercontent.com/saiarlen/deployCP/main/scripts/linux/install-remote.sh | sudo bash
Fresh Install

Install the latest release.

curl -fsSL https://raw.githubusercontent.com/saiarlen/deployCP/main/scripts/linux/install-remote.sh | sudo bash
This is the primary install flow.

No manual binary download or extract step is required for the normal operator path.

Pinned Version

Install a specific tagged release.

curl -fsSL https://raw.githubusercontent.com/saiarlen/deployCP/main/scripts/linux/install-remote.sh | sudo DEPLOYCP_VERSION=v1.2.3 bash

Use this when you want a controlled rollout instead of the latest published release.

Update

Update an existing host with the same bootstrap entrypoint.

curl -fsSL https://raw.githubusercontent.com/saiarlen/deployCP/main/scripts/linux/install-remote.sh | sudo bash -s -- --update

The updater downloads the selected release bundle, stages the new binary, runs the packaged update script, reconciles managed state, and verifies the host afterward.

Bootstrap Steps

What the packaged installer does.

01

Validates root and host prerequisites

Detects distro family, installs required packages, resolves service names and binary paths, and checks the system can run live Linux operations.

02

Creates the managed DeployCP layout

Creates the deploycp OS user, home directory, core layout, runtime state folders, generated config directories, logs, metadata, backups, temp paths, and helper script locations.

03

Installs and starts DeployCP

Writes the environment file, stages the release binary into /home/deploycp/core/bin/deploycp, installs deploycp.service, runs host bootstrap, and starts the service.

04

Runs host verification

Performs post-install verification so the operator can fix missing host-level prerequisites before creating live platforms.

After Install

Immediate checks and values to set.

Service health

sudo systemctl status deploycp --no-pager

Host verification

sudo /home/deploycp/core/bin/deploycp verify-host

Environment file

Review /home/deploycp/core/.env and set the real APP_BASE_URL, bootstrap admin credentials, and local DB admin passwords if managed DB provisioning is required.

Uninstall

Remove DeployCP from the host.

This removes live managed resources.

The uninstall flow stops the service, tears down DeployCP-managed host state, removes the deploycp user, and deletes the managed home directory.

sudo /home/deploycp/core/scripts/linux/uninstall.sh
Fallback Only

Manual tarball install when remote bootstrap is not allowed.

This is a fallback path, not the primary public install recommendation.

tar -xzf deploycp-v1.2.3-linux-amd64.tar.gz
cd deploycp-v1.2.3-linux-amd64
sudo ./scripts/linux/install.sh
